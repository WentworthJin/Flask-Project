Swift Function

Swift function is an independent block of code used to complete a specific task.

Swift uses a unified syntax to represent simple C language style functions to complex Objective-C language style methods.

  • Function declaration: Tell the compiler the name of the function, return type and parameters.
  • Function definition: Provides the entity of the function.

Swift functions include parameter types and return value types:

Function definition

Swift defines functions using the keyword func .

When defining a function, you can specify one or more input parameters and a return value type.

Every function has a function name to describe its function. This function is called by the function name and the parameter value of the corresponding type. The order in which the parameters of the function are passed must be the same as the parameter list.

The order in which the actual parameters of the function are passed must be the same as the formal parameter list, and the return value type of the function is defined after -> .

Grammar

func funcname(parameter) -> returntype
    {
       Statement1
       Statement2
       ……
       Statement N
       return parameters
    }

Instance

Below we define a function named youj, the data type of the formal parameter is String, and the return value is also String:

import Cocoa
    
    func youj(site: String) -> String {
        return site
    }
    print(youj("google.com"))

The output of the above program execution is:

google.com

Function call

We can call the function by the function name and the parameter value of the corresponding type, and the order of the parameter passing of the function must be the same as the parameter list.

Below we define a function named youj. The data type of the formal parameter site is String. After that, the actual parameter passed by the function must also be of type String. After the actual parameter is passed into the function body, it will return directly. Is String. import Cocoa 

import Cocoa
    
    func youj(site: String) -> String {
        return site
    }
    print(youj("google.com"))

The output of the above program execution is:

google.com

Function parameter

Functions can accept one or more parameters, and we can also use tuples to pass one or more parameters to the function:

import Cocoa
    
    func mult(no1: Int, no2: Int) -> Int {
       return no1*no2
    }
    print(mult(2, no2:20))
    print(mult(3, no2:15))
    print(mult(4, no2:30))

The output of the above program execution is:

40
    45
    120

Tuples as function return values

The function return value type can be string, integer, floating point, etc.

Tuples are similar to arrays, except that the elements in a tuple can be of any type, using parentheses.

You can use the tuple type to return multiple values from a function as a composite value.

In the following example, a function named minMax(_:) is defined, which is used to find the minimum and maximum values in an Int array.

import Cocoa
    
    func minMax(array: [Int]) -> (min: Int, max: Int) {
        var currentMin = array[0]
        var currentMax = array[0]
        for value in array[1..<array.count] {
            if value < currentMin {
                currentMin = value
            } else if value > currentMax {
                currentMax = value
            }
        }
        return (currentMin, currentMax)
    }
    
    let bounds = minMax([8, -6, 2, 109, 3, 71])
    print("The min: \(bounds.min), The max: \(bounds.max)")

The output of the above program execution is:

The min: -6, The max: 109

Caution:

To safely handle the "empty array" problem, minMax(_:)rewrite the function to use an optional tuple return type, and return when the array is empty nil:

if array.isEmpty {return nil}

Function parameter name

Function parameters have an external parameter name and a local parameter name.

Local parameter name

Local parameter names are used internally in the implementation of the function.

func sample(number: Int) {
       println(number)
    }

In the above example, number is a local parameter name, which can only be used in the function body.

External parameter name

You can specify the external parameter name before the local parameter name, separated by spaces. The external parameter name is used for the parameters passed to the function when the function is called.

import Cocoa
    
    func pow(firstArg a: Int, secondArg b: Int) -> Int {
       var res = a
       for _ in 1..<b {
          res = res * a
       }
       print(res)
       return res
    }
    pow(firstArg:5, secondArg:3)

The output of the above program execution is:

125

Variable parameter

Variable parameters can accept zero or more values. When calling a function, you can use variable parameters to specify function parameters, the number of which is uncertain.

Variable parameters are defined by adding (...) after the variable type name.

import Cocoa
    
    func vari<N>(members: N...){
        for i in members {
            print(i)
        }
    }
    vari(4,3,5)
    vari(4.5, 3.1, 5.6)
    vari("Google", "Facebook", "Amazon")

The output of the above program execution is:

4
    3
    5
    4.5
    3.1
    5.6
    Google
    Facebook
    Amazon

Constants, variables and I/O parameters

Generally, the parameters defined in the function are constant parameters by default, that is, you can only query and use this parameter, and you cannot change its value.

If you want to declare a variable parameter, you can add the inout keyword before the parameter definition, so that you can change the value of the parameter.

Generally, the default parameter passing is called by value, not by reference. Therefore, the incoming parameters are changed within the function, and it does not affect the original parameter. What is passed in is just a copy of this parameter.

When the incoming parameter is used as an input and output parameter, you need to add an ampersand before the parameter name, which means that the value can be modified by the function.

    let temporaryA = a
    
    a = b

}
